## Story
**As a** developer,
**I want** synchronous S3 storage calls wrapped in async to avoid blocking the event loop,
**so that** the API can handle concurrent requests efficiently even during storage operations.

## Dev Notes

### AsyncS3Client Pattern
```python
import asyncio
from minio import Minio

class AsyncS3Client:
    def __init__(self, sync_client: Minio):
        self._client = sync_client

    async def list_objects(self, bucket: str, prefix: str = "", recursive: bool = False):
        return await asyncio.to_thread(
            lambda: list(self._client.list_objects(bucket, prefix=prefix, recursive=recursive))
        )

    async def put_object(self, bucket, key, data, length, content_type="application/octet-stream"):
        return await asyncio.to_thread(
            self._client.put_object, bucket, key, data, length, content_type=content_type
        )

    async def get_object(self, bucket, key, **kwargs):
        return await asyncio.to_thread(self._client.get_object, bucket, key, **kwargs)

    async def stat_object(self, bucket, key):
        return await asyncio.to_thread(self._client.stat_object, bucket, key)

    async def remove_object(self, bucket, key):
        return await asyncio.to_thread(self._client.remove_object, bucket, key)
    # ... etc
```

### Key Difference: list_objects
The sync `list_objects` returns a lazy iterator. Must consume it inside the thread:
```python
async def list_objects(self, ...):
    return await asyncio.to_thread(
        lambda: list(self._client.list_objects(...))  # consume iterator in thread
    )
```

- Only change `async def` endpoints — leave `def` (sync) endpoints alone
- Queue worker runs in its own process, not async — keeps sync client
- `get_object` returns a urllib3.HTTPResponse — the response object itself is sync (reads body later)
- For streaming responses, the read happens in the endpoint — may need `to_thread` there too
- Don't change function signatures — internal refactor only
