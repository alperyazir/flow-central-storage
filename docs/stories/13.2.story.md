# Story 13.2: Data Migration Tooling

## Status
Approved

## Story
**As a** system administrator,
**I want** a migration script to move existing data from MinIO to SeaweedFS,
**so that** all publisher content, app templates, teacher materials, and trash are preserved.

## Acceptance Criteria
1. Migration script copies all objects from all 4 MinIO buckets to SeaweedFS
2. Object counts match between source (MinIO) and destination (SeaweedFS) post-migration
3. Migration can run with both MinIO and SeaweedFS running simultaneously (zero downtime)
4. Rollback procedure documented (keep MinIO data until verified)
5. Migration script is idempotent (safe to re-run)

- [ ] Task 1: Create migration script (AC: 1, 3, 5)
  - [ ] Create `infrastructure/scripts/migrate-minio-to-seaweedfs.sh`
  - [ ] Use `rclone` to sync between two S3 endpoints
  - [ ] Configure source (MinIO at localhost:9000) and destination (SeaweedFS at localhost:8333)
  - [ ] Migrate buckets in order: publishers, apps, teachers, trash
  - [ ] Script should be idempotent — `rclone sync` skips existing identical files
  - [ ] Add progress output per bucket

  - [ ] Add migration section to `docs/deployment-runbook.md`
  - [ ] Document: prerequisites, running the script, verification, rollback
  - [ ] Rollback = point config back to MinIO endpoint, MinIO data untouched

  - [ ] Document verification results in completion notes

## Dev Notes

### Migration Approach
```bash
# Install rclone if not available
# Configure two remotes:
rclone config create minio-old s3 \
  provider=Minio endpoint=http://localhost:9000 \
  access_key_id=flow_minio secret_access_key=flow_minio_secret

rclone config create seaweedfs-new s3 \
  provider=Other endpoint=http://localhost:8333 \
  access_key_id=admin secret_access_key=admin

# Sync each bucket
for bucket in publishers apps teachers trash; do
  rclone sync minio-old:$bucket seaweedfs-new:$bucket --progress
done
```

### Buckets to Migrate
- `publishers` — book archives, logos, materials (largest bucket)
- `apps` — standalone app templates
- `teachers` — teacher-uploaded materials
- `trash` — soft-deleted items

### Prerequisites
- Both MinIO (port 9000) and SeaweedFS (port 8333) running
- `rclone` installed (`brew install rclone` or `apt install rclone`)
- Buckets must exist in SeaweedFS (run init script from Story 13.1 first)

### Important
- MinIO data is NOT deleted by this script — it's a copy, not a move
- `rclone sync` is idempotent — re-running only copies new/changed files
- For large datasets, migration time scales linearly with total size

## Testing
