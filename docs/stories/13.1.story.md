## Dev Notes

### SeaweedFS Docker Compose Pattern
```yaml
seaweedfs-master:
  image: chrislusf/seaweedfs:latest
  command: "master -ip=seaweedfs-master -ip.bind=0.0.0.0"
  ports:
    - "9333:9333"   # master API
  volumes:
    - seaweedfs_data:/data

seaweedfs-volume:
  image: chrislusf/seaweedfs:latest
  command: "volume -mserver=seaweedfs-master:9333 -ip.bind=0.0.0.0 -port=8080"
  depends_on:
    - seaweedfs-master

seaweedfs-filer:
  image: chrislusf/seaweedfs:latest
  command: "filer -master=seaweedfs-master:9333 -ip.bind=0.0.0.0"
  ports:
    - "8888:8888"   # filer UI
  depends_on:
    - seaweedfs-master
    - seaweedfs-volume

seaweedfs-s3:
  image: chrislusf/seaweedfs:latest
  command: "s3 -filer=seaweedfs-filer:8888 -ip.bind=0.0.0.0 -port=8333"
  ports:
    - "8333:8333"   # S3 API
  depends_on:
    - seaweedfs-filer
```

### Current MinIO Config (config.py)
- `minio_endpoint: str = "localhost:9000"` — change to `seaweedfs-s3:8333`
- `minio_external_url: str = "http://localhost:9000"` — change to `http://localhost:8333`
- `minio_access_key / minio_secret_key` — SeaweedFS S3 uses its own auth config
- `minio_secure: bool = False` — keep as-is

### Key Files
- `infrastructure/docker-compose.yml` — main service definitions
- `apps/api/app/core/config.py` — storage endpoint config
- `apps/api/app/services/minio.py` — client creation (NO changes needed — uses standard S3)
- `apps/api/app/scripts/init_minio.py` — bucket init (NO changes needed)
- `apps/api/.env.example` — env var documentation
- `apps/api/app/routers/health.py` — health check (NO changes needed — uses Minio SDK which talks S3)

### Important
- **Do NOT rename Python code** — `minio` SDK, `get_minio_client()`, `minio_endpoint` config vars all stay. They're just S3 client wrappers.
- SeaweedFS S3 gateway supports `access_key` and `secret_key` via `-config` flag or defaults
- If SeaweedFS S3 runs without auth config, any key works. For dev this is fine.
