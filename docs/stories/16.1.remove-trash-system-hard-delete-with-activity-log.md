# Story 16.1: Remove Trash System — Hard Delete with Activity Log

## Status: In Progress

## Story
As an admin, I want book deletions to be permanent (no trash/restore cycle) and I want upload/delete operations to show real-time progress in a non-blocking activity log panel, so that I can continue working while operations complete.

## Acceptance Criteria

### Backend
- [ ] `DELETE /books/{book_id}` permanently deletes files from R2 and removes the DB record (no trash bucket, no ARCHIVED status)
- [ ] New `DELETE /books/{book_id}/async` endpoint returns job_id immediately (202), deletes files in background with progress tracking
- [ ] New `GET /books/delete-status/{job_id}` endpoint returns deletion progress (reuses Redis progress pattern from upload)
- [ ] Trash-related endpoints remain functional for app/teacher materials but books skip trash entirely
- [ ] Webhook `BOOK_DELETED` still fires after successful hard delete
- [ ] DB connection released before slow R2 operations (existing pattern)

### Frontend — Activity Log Panel
- [ ] Floating panel in bottom-right corner (Google Drive style) showing operation history
- [ ] Panel shows last 10 operations (uploads + deletes) with:
  - Operation type icon (upload ↑ / delete ✕)
  - Book name
  - Timestamp (HH:MM)
  - Progress bar (during operation)
  - Status: ✓ (success) or ✗ (failed) with color coding
- [ ] Panel is collapsible/expandable
- [ ] Operations are non-blocking — user can navigate and work while they run
- [ ] Panel persists across page navigation (lives in layout, not page)

### Frontend — Book Management Cleanup
- [ ] Remove upload button from All Books page (`/books`)
- [ ] Book upload only available from Publisher Detail page (`/publishers/:id`)
- [ ] Book delete only available from Publisher Detail page
- [ ] Remove Trash page from sidebar navigation
- [ ] Remove Trash page component

## Dev Notes

### Backend Changes

**Modified Files:**
- `apps/api/app/routers/books.py`
  - Change `soft_delete_book` → `hard_delete_book`: skip `move_prefix_to_trash()`, instead call new `delete_prefix_directly()`, then `_book_repository.delete()` instead of `archive()`
  - Add async delete endpoint with progress tracking (mirror upload-async pattern)
  - Add delete-status endpoint (mirror upload-status)
- `apps/api/app/services/storage.py`
  - Add `delete_prefix_directly()`: list objects under prefix → `remove_object()` each → return DeletionReport
- `apps/api/app/services/cache.py`
  - Add `set_deletion_progress()` / `get_deletion_progress()` / `delete_deletion_progress()` (same Redis pattern as upload)

**Unchanged:**
- Trash endpoints in `storage.py` — keep for apps/teacher materials
- `move_prefix_to_trash()` — keep for non-book use cases

### Frontend Changes

**New Component:**
- `src/components/ActivityLogPanel.tsx` — Floating bottom-right panel
  - Uses Zustand store for operation state (persists across navigation)
  - Polls status endpoints for active operations
  - Shows last 10 entries with auto-cleanup

**New Store:**
- `src/stores/operations.ts` — Zustand store
  ```typescript
  interface Operation {
    id: string
    type: 'upload' | 'delete'
    bookName: string
    status: 'pending' | 'in_progress' | 'completed' | 'failed'
    progress: number
    timestamp: Date
    error?: string
  }
  ```

**Modified Files:**
- `src/layouts/MainLayout.tsx` — Add `<ActivityLogPanel />` 
- `src/pages/PublisherDetail.tsx` — Add delete button to books table, integrate with async delete + activity log
- `src/pages/Books.tsx` — Remove upload button, remove delete button
- `src/components/AppSidebar.tsx` — Remove Trash navigation item
- `src/components/PublisherUploadDialog.tsx` — Integrate with activity log store instead of blocking dialog

**Removed Files:**
- `src/pages/Trash.tsx`

### API Contract

**New: Async Delete**
```
DELETE /books/{book_id}/async
Response 202: { "job_id": "uuid", "status": "accepted" }
```

**New: Delete Status**
```
GET /books/delete-status/{job_id}
Response 200: {
  "progress": 75,
  "step": "deleting",
  "detail": "120/160 files",
  "error": null
}
```
